Chapter 22: Building a Text-to-SQL Layer on Your Schema

Natural language database querying has moved from research curiosity to production requirement. When your product manager asks "why can't users just ask questions about their data?", the answer used to be "it's too hard." With modern LLMs, the hard part shifts from generating syntactically valid SQL to generating *semantically correct* SQL — queries that do what the user actually meant, against your specific schema, with proper filters, joins, and aggregations. This chapter builds a complete Text-to-SQL layer: schema introspection and serialization, prompt engineering for SQL generation, result validation, query execution safety, caching, and a FastAPI endpoint that ties it all together. PostgreSQL's rich metadata system, row-level security, and JSONB capabilities make it uniquely suited to be both the query target and the operational backbone of this architecture.

---

The Anatomy of a Text-to-SQL System

Text-to-SQL is not a single model call. It's a pipeline with several moving parts, each of which can fail independently:

1. Schema understanding — the LLM needs to know your table names, column names, types, relationships, and constraints. Without this, it generates plausible-looking SQL that references columns that don't exist.

2. Intent extraction — "show me the top customers last month" contains temporal ambiguity, aggregation intent, and an undefined metric ("top" by revenue? order count?).

3. SQL generation — the LLM produces a candidate query.

4. SQL validation — syntactic check, safety check (no DDL, no writes), and optionally a dry-run EXPLAIN.

5. Result rendering — turning rows into something a user can read.

6. Feedback loop — storing query history so you can improve your prompts and catch systematic errors.

PostgreSQL sits at the center of every step. Your schema lives there. Your users' data lives there. And your query history, embeddings for few-shot examples, and audit logs live there too.

The architecture looks like this:

User question
     │
     ▼
Schema context builder (pg_catalog, information_schema)
     │
     ▼
Few-shot retriever (pgvector similarity search on past Q→SQL pairs)
     │
     ▼
LLM (Claude or GPT-4) generates SQL
     │
     ▼
Validator (syntax, safety, EXPLAIN)
     │
     ▼
Executor (read-only role, RLS enforced)
     │
     ▼
Result formatter
     │
     ▼
Audit log (query_history table)

Before writing any Python, get this schema in place.

Schema Design: Metadata, History, and Few-Shot Storage

A Text-to-SQL system needs its own operational tables. These live in a dedicated schema alongside your application data.

-- Schema for the Text-to-SQL system itself
CREATE SCHEMA IF NOT EXISTS text2sql;

-- Cache of schema introspection results
-- Rebuilt on schema changes via pg_notify or periodic refresh
CREATE TABLE text2sql.schema_snapshots (
    id               BIGSERIAL PRIMARY KEY,
    snapshot_hash    TEXT NOT NULL,
    schema_names     TEXT[] NOT NULL,
    serialized_ddl   TEXT NOT NULL,           -- compact DDL string sent to LLM
    table_count      INTEGER NOT NULL,
    column_count     INTEGER NOT NULL,
    created_at       TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    is_current       BOOLEAN NOT NULL DEFAULT TRUE
);

CREATE UNIQUE INDEX idx_schema_snapshots_current
    ON text2sql.schema_snapshots (is_current)
    WHERE is_current = TRUE;

-- Query history: every question asked, every SQL generated
CREATE TABLE text2sql.query_history (
    id               BIGSERIAL PRIMARY KEY,
    session_id       UUID NOT NULL,
    user_id          TEXT,                    -- application-level user identifier
    question         TEXT NOT NULL,
    generated_sql    TEXT,
    normalized_sql   TEXT,                    -- lowercased, whitespace-normalized
    execution_status TEXT NOT NULL            -- 'success', 'sql_error', 'safety_blocked', 'llm_error'
        CHECK (execution_status IN ('success', 'sql_error', 'safety_blocked', 'llm_error', 'timeout')),
    row_count        INTEGER,
    execution_ms     INTEGER,
    error_message    TEXT,
    model_used       TEXT NOT NULL,
    prompt_tokens    INTEGER,
    completion_tokens INTEGER,
    schema_snapshot_id BIGINT REFERENCES text2sql.schema_snapshots(id),
    created_at       TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_query_history_session   ON text2sql.query_history (session_id);
CREATE INDEX idx_query_history_user      ON text2sql.query_history (user_id);
CREATE INDEX idx_query_history_status    ON text2sql.query_history (execution_status);
CREATE INDEX idx_query_history_created   ON text2sql.query_history (created_at DESC);

-- Few-shot examples: verified Q→SQL pairs used in prompts
-- The embedding allows semantic retrieval of relevant examples
CREATE EXTENSION IF NOT EXISTS vector;

CREATE TABLE text2sql.few_shot_examples (
    id               BIGSERIAL PRIMARY KEY,
    question         TEXT NOT NULL,
    sql_query        TEXT NOT NULL,
    explanation      TEXT,                    -- why this SQL answers this question
    tags             TEXT[],                  -- 'aggregation', 'time_filter', 'join', etc.
    is_verified      BOOLEAN NOT NULL DEFAULT FALSE,
    source           TEXT NOT NULL DEFAULT 'manual'
        CHECK (source IN ('manual', 'promoted_from_history', 'synthetic')),
    question_embedding VECTOR(1536),          -- OpenAI text-embedding-3-small
    upvotes          INTEGER NOT NULL DEFAULT 0,
    downvotes        INTEGER NOT NULL DEFAULT 0,
    created_at       TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at       TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_few_shot_embedding
    ON text2sql.few_shot_examples
    USING hnsw (question_embedding vector_cosine_ops)
    WITH (m = 16, ef_construction = 64);

CREATE INDEX idx_few_shot_verified
    ON text2sql.few_shot_examples (is_verified)
    WHERE is_verified = TRUE;

-- Column metadata: human-readable descriptions to supplement information_schema
-- Critical for columns with non-obvious names like "rev_adj_12m" or "status_cd"
CREATE TABLE text2sql.column_descriptions (
    id               BIGSERIAL PRIMARY KEY,
    schema_name      TEXT NOT NULL,
    table_name       TEXT NOT NULL,
    column_name      TEXT NOT NULL,
    description      TEXT NOT NULL,
    example_values   TEXT[],                  -- 2-3 representative values shown in prompt
    is_sensitive     BOOLEAN NOT NULL DEFAULT FALSE,  -- exclude from LLM context
    UNIQUE (schema_name, table_name, column_name)
);

-- Audit log: what queries actually ran, separate from query_history
-- Append-only, never updated
CREATE TABLE text2sql.execution_audit (
    id               BIGSERIAL PRIMARY KEY,
    query_history_id BIGINT REFERENCES text2sql.query_history(id),
    executed_sql     TEXT NOT NULL,
    execution_plan   JSONB,                   -- from EXPLAIN (FORMAT JSON)
    rows_returned    INTEGER,
    execution_ms     INTEGER,
    pg_user          TEXT NOT NULL DEFAULT current_user,
    client_addr      INET,
    executed_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
) PARTITION BY RANGE (executed_at);

CREATE TABLE text2sql.execution_audit_2024
    PARTITION OF text2sql.execution_audit
    FOR VALUES FROM ('2024-01-01') TO ('2025-01-01');

CREATE TABLE text2sql.execution_audit_2025
    PARTITION OF text2sql.execution_audit
    FOR VALUES FROM ('2025-01-01') TO ('2026-01-01');

-- Read-only role for query execution
-- This role is what the Text-to-SQL runner connects as
DO $$
BEGIN
    IF NOT EXISTS (SELECT 1 FROM pg_roles WHERE rolname = 'text2sql_reader') THEN
        CREATE ROLE text2sql_reader NOLOGIN;
    END IF;
END
$$;

-- Grant SELECT on application schemas only — never text2sql schema itself
GRANT USAGE ON SCHEMA public TO text2sql_reader;
GRANT SELECT ON ALL TABLES IN SCHEMA public TO text2sql_reader;
ALTER DEFAULT PRIVILEGES IN SCHEMA public
    GRANT SELECT ON TABLES TO text2sql_reader;

-- Create login user that uses the reader role
DO $$
BEGIN
    IF NOT EXISTS (SELECT 1 FROM pg_roles WHERE rolname = 'text2sql_app') THEN
        CREATE USER text2sql_app PASSWORD 'use_a_real_secret_here';
    END IF;
END
$$;

GRANT text2sql_reader TO text2sql_app;

The separation between query_history (what users asked and what SQL was generated) and execution_audit (what actually executed against the database) is intentional. Query history is writeable by the application. The audit table is append-only and should have a separate, more privileged writer.

Schema Introspection: Giving the LLM Your DDL

The most common failure mode in Text-to-SQL is the LLM hallucinating column names or table structures it wasn't told about. The fix is thorough schema serialization — extracting your actual DDL from pg_catalog and information_schema and including it in every prompt.

-- Function to extract compact DDL for Text-to-SQL prompts
-- Returns a text block suitable for direct inclusion in an LLM prompt
CREATE OR REPLACE FUNCTION text2sql.get_schema_ddl(
    target_schemas TEXT[] DEFAULT ARRAY['public']
)
RETURNS TEXT
LANGUAGE plpgsql
STABLE
AS $$
DECLARE
    result TEXT := '';
    rec RECORD;
    col_rec RECORD;
    fk_rec RECORD;
    table_ddl TEXT;
    col_list TEXT;
BEGIN
    FOR rec IN
        SELECT
            n.nspname AS schema_name,
            c.relname AS table_name,
            obj_description(c.oid, 'pg_class') AS table_comment
        FROM pg_class c
        JOIN pg_namespace n ON n.oid = c.relnamespace
        WHERE n.nspname = ANY(target_schemas)
          AND c.relkind IN ('r', 'v', 'm')  -- tables, views, materialized views
          AND c.relname NOT LIKE 'pg_%'
        ORDER BY n.nspname, c.relname
    LOOP
        col_list := '';

        FOR col_rec IN
            SELECT
                a.attname AS column_name,
                pg_catalog.format_type(a.atttypid, a.atttypmod) AS data_type,
                a.attnotnull AS not_null,
                pg_get_expr(ad.adbin, ad.adrelid) AS column_default,
                col_description(a.attrelid, a.attnum) AS col_comment,
                -- Check if this column is part of a primary key
                EXISTS (
                    SELECT 1 FROM pg_constraint con
                    JOIN pg_attribute att ON att.attrelid = con.conrelid
                        AND att.attnum = ANY(con.conkey)
                    WHERE con.conrelid = a.attrelid
                      AND con.contype = 'p'
                      AND att.attname = a.attname
                ) AS is_pk,
                -- Check for text2sql column description
                cd.description AS custom_description,
                cd.example_values,
                cd.is_sensitive
            FROM pg_attribute a
            LEFT JOIN pg_attrdef ad ON ad.adrelid = a.attrelid AND ad.adnum = a.attnum
            LEFT JOIN text2sql.column_descriptions cd
                ON cd.schema_name = rec.schema_name
                AND cd.table_name = rec.table_name
                AND cd.column_name = a.attname
            WHERE a.attrelid = (
                SELECT oid FROM pg_class
                WHERE relname = rec.table_name
                  AND relnamespace = (SELECT oid FROM pg_namespace WHERE nspname = rec.schema_name)
            )
              AND a.attnum > 0
              AND NOT a.attisdropped
              AND (cd.is_sensitive IS NULL OR cd.is_sensitive = FALSE)
            ORDER BY a.attnum
        LOOP
            col_list := col_list ||
                '  ' || col_rec.column_name ||
                ' ' || col_rec.data_type ||
                CASE WHEN col_rec.is_pk THEN ' PRIMARY KEY' ELSE '' END ||
                CASE WHEN col_rec.not_null AND NOT col_rec.is_pk THEN ' NOT NULL' ELSE '' END ||
                CASE
                    WHEN col_rec.custom_description IS NOT NULL THEN
                        ' -- ' || col_rec.custom_description ||
                        CASE
                            WHEN col_rec.example_values IS NOT NULL THEN
                                ' (e.g., ' || array_to_string(col_rec.example_values, ', ') || ')'
                            ELSE ''
                        END
                    WHEN col_rec.col_comment IS NOT NULL THEN
                        ' -- ' || col_rec.col_comment
                    ELSE ''
                END ||
                E',\n';
        END LOOP;

        -- Remove trailing comma+newline from last column
        col_list := rtrim(col_list, E',\n');

        table_ddl := 'CREATE TABLE ' || rec.schema_name || '.' || rec.table_name || ' (\n' ||
                     col_list || E'\n);\n';

        IF rec.table_comment IS NOT NULL THEN
            table_ddl := '-- ' || rec.table_comment || E'\n' || table_ddl;
        END IF;

        result := result || table_ddl || E'\n';
    END LOOP;

    -- Append foreign key relationships
    result := result || E'\n-- FOREIGN KEY RELATIONSHIPS\n';

    FOR fk_rec IN
        SELECT
            tc.table_schema,
            tc.table_name,
            kcu.column_name,
            ccu.table_schema AS foreign_table_schema,
            ccu.table_name AS foreign_table_name,
            ccu.column_name AS foreign_column_name,
            tc.constraint_name
        FROM information_schema.table_constraints tc
        JOIN information_schema.key_column_usage kcu
            ON tc.constraint_name = kcu.constraint_name
            AND tc.table_schema = kcu.table_schema
        JOIN information_schema.constraint_column_usage ccu
            ON ccu.constraint_name = tc.constraint_name
        WHERE tc.constraint_type = 'FOREIGN KEY'
          AND tc.table_schema = ANY(target_schemas)
        ORDER BY tc.table_name, kcu.column_name
    LOOP
        result := result ||
            '-- ' || fk_rec.table_name || '.' || fk_rec.column_name ||
            ' → ' || fk_rec.foreign_table_name || '.' || fk_rec.foreign_column_name || E'\n';
    END LOOP;

    RETURN result;
END;
$$;

-- Quick view for monitoring schema snapshot freshness
CREATE OR REPLACE VIEW text2sql.schema_status AS
SELECT
    ss.id,
    ss.snapshot_hash,
    ss.table_count,
    ss.column_count,
    ss.created_at,
    ss.is_current,
    qh.question_count,
    qh.success_rate
FROM text2sql.schema_snapshots ss
LEFT JOIN LATERAL (
    SELECT
        COUNT(*) AS question_count,
        ROUND(
            100.0 * SUM(CASE WHEN execution_status = 'success' THEN 1 ELSE 0 END) / COUNT(*),
            1
        ) AS success_rate
    FROM text2sql.query_history
    WHERE schema_snapshot_id = ss.id
) qh ON TRUE;

Now call this from Python to get the DDL string that goes into your prompt:

# schema_builder.py — Schema introspection and serialization for Text-to-SQL prompts

import hashlib
import asyncpg
import asyncio
from dataclasses import dataclass
from typing import Optional
import logging

logger = logging.getLogger(__name__)


@dataclass
class SchemaSnapshot:
    snapshot_id: int
    snapshot_hash: str
    serialized_ddl: str
    table_count: int
    column_count: int


async def get_or_refresh_schema_snapshot(
    pool: asyncpg.Pool,
    target_schemas: list[str] = None,
    force_refresh: bool = False
) -> SchemaSnapshot:
    """
    Returns the current schema snapshot, refreshing if the schema has changed.
    Uses a hash of the raw DDL to detect changes without storing redundant snapshots.
    """
    if target_schemas is None:
        target_schemas = ["public"]

    async with pool.acquire() as conn:
        # Get fresh DDL from pg_catalog
        raw_ddl = await conn.fetchval(
            "SELECT text2sql.get_schema_ddl($1::text[])",
            target_schemas
        )

        # Count tables and columns for metadata
        counts = await conn.fetchrow("""
            SELECT
                COUNT(DISTINCT table_name)::int AS table_count,
                COUNT(*)::int AS column_count
            FROM information_schema.columns
            WHERE table_schema = ANY($1::text[])
        """, target_schemas)

        current_hash = hashlib.sha256(raw_ddl.encode()).hexdigest()[:16]

        # Check if current snapshot matches
        if not force_refresh:
            existing = await conn.fetchrow("""
                SELECT id, snapshot_hash, serialized_ddl, table_count, column_count
                FROM text2sql.schema_snapshots
                WHERE is_current = TRUE
            """)

            if existing and existing["snapshot_hash"] == current_hash:
                return SchemaSnapshot(
                    snapshot_id=existing["id"],
                    snapshot_hash=existing["snapshot_hash"],
                    serialized_ddl=existing["serialized_ddl"],
                    table_count=existing["table_count"],
                    column_count=existing["column_count"]
                )

        # Schema has changed or no snapshot exists — create new one
        async with conn.transaction():
            # Mark all existing snapshots as not current
            await conn.execute("""
                UPDATE text2sql.schema_snapshots
                SET is_current = FALSE
                WHERE is_current = TRUE
            """)

            # Insert new snapshot
            new_id = await conn.fetchval("""
                INSERT INTO text2sql.schema_snapshots
                    (snapshot_hash, schema_names, serialized_ddl, table_count, column_count, is_current)
                VALUES ($1, $2, $3, $4, $5, TRUE)
                RETURNING id
            """, current_hash, target_schemas, raw_ddl,
                counts["table_count"], counts["column_count"])

            logger.info(
                f"Schema snapshot refreshed: {current_hash} "
                f"({counts['table_count']} tables, {counts['column_count']} columns)"
            )

            return SchemaSnapshot(
                snapshot_id=new_id,
                snapshot_hash=current_hash,
                serialized_ddl=raw_ddl,
                table_count=counts["table_count"],
                column_count=counts["column_count"]
            )


async def get_relevant_few_shots(
    pool: asyncpg.Pool,
    question_embedding: list[float],
    limit: int = 5
) -> list[dict]:
    """
    Retrieve semantically similar verified Q→SQL pairs for few-shot prompting.
    Uses cosine similarity on the question_embedding column.
    """
    async with pool.acquire() as conn:
        rows = await conn.fetch("""
            SELECT
                question,
                sql_query,
                explanation,
                1 - (question_embedding <=> $1::vector) AS similarity
            FROM text2sql.few_shot_examples
            WHERE is_verified = TRUE
              AND question_embedding IS NOT NULL
            ORDER BY question_embedding <=> $1::vector
            LIMIT $2
        """, question_embedding, limit)

        return [dict(row) for row in rows]

SQL Generation: Prompt Engineering and the LLM Call

The quality of your SQL output depends heavily on prompt structure. You need to tell the LLM: what dialect it's working in, what your schema looks like, relevant examples, and what constraints apply. You also need to handle the case where the question can't be answered from the available schema.
