Chapter 10: User Preference Storage for Personalized AI

Personalization is what separates a generic AI assistant from one that users actually want to come back to. When your application remembers that a user prefers concise responses, works primarily in Python, operates in the Pacific timezone, and gets frustrated by overly formal language, every interaction improves. This chapter covers how to design and implement a complete user preference system for AI applications — one that goes beyond a simple key-value store to capture explicit settings, inferred behavioral signals, semantic preferences expressed as vectors, and the machinery to make all of it available at inference time without destroying your latency budget.

---

The Anatomy of AI User Preferences

User preferences in AI applications are not a single thing. They exist at multiple layers, each contributing differently to how your model behaves.

Explicit preferences are settings the user consciously sets: "Respond in Spanish," "Keep answers under 200 words," "Don't use markdown formatting." These are discrete, structured, and easy to act on. They belong in typed columns with strict constraints.

Implicit preferences are inferred from behavior: a user who consistently upvotes short answers probably wants brevity even if they never said so. A user who rephrases questions about code into more specific forms after getting broad answers is signaling they want precision. These are statistical signals extracted from interaction logs.

Semantic preferences are the most interesting for AI systems. They capture concepts that don't reduce cleanly to a flag or a number. "This user likes analogies from physics" or "prefers examples from the financial domain" are preferences that live in embedding space and can be retrieved by similarity against incoming queries.

Session-level context is temporary preference — what the user needs right now, in this conversation, based on what they've said in the last ten turns. This is your conversation history feeding into in-context personalization.

The reason PostgreSQL handles this well is that it speaks all four of these languages natively. JSONB handles arbitrary structured settings. Numeric aggregations and window functions extract implicit signals from logs. pgvector stores semantic preference embeddings. And foreign keys tie all of it to a single users table that your security policies can anchor to.

A Redis-backed cache layer gives you sub-millisecond preference retrieval so that adding personalization doesn't add perceptible latency to inference calls.

Schema Design: Layered Preference Storage

The schema needs to support all four preference types without collapsing them into an unmanageable blob. Here is a production-ready design:

-- Core user table (referenced by all preference tables)
CREATE TABLE users (
    user_id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    external_id     TEXT UNIQUE NOT NULL,          -- maps to your auth provider
    email           TEXT,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
    last_active_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
    is_active       BOOLEAN NOT NULL DEFAULT true
);

-- Explicit preferences: typed, constrained, version-tracked
CREATE TABLE user_explicit_preferences (
    preference_id       UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id             UUID NOT NULL REFERENCES users(user_id) ON DELETE CASCADE,
    response_language   TEXT NOT NULL DEFAULT 'en',
    response_length     TEXT NOT NULL DEFAULT 'medium'
                            CHECK (response_length IN ('brief', 'medium', 'detailed')),
    tone                TEXT NOT NULL DEFAULT 'neutral'
                            CHECK (tone IN ('formal', 'neutral', 'casual')),
    code_language       TEXT,                      -- preferred programming language
    use_markdown        BOOLEAN NOT NULL DEFAULT true,
    max_response_tokens INTEGER CHECK (max_response_tokens BETWEEN 50 AND 8000),
    domain_focus        TEXT[],                    -- e.g. ARRAY['finance', 'healthcare']
    avoid_topics        TEXT[],                    -- topics to steer clear of
    custom_instructions TEXT,                      -- free-form system prompt addendum
    updated_at          TIMESTAMPTZ NOT NULL DEFAULT now(),
    version             INTEGER NOT NULL DEFAULT 1,
    UNIQUE (user_id)
);

-- Implicit preferences: aggregated signals from interaction logs
CREATE TABLE user_implicit_preferences (
    user_id                 UUID PRIMARY KEY REFERENCES users(user_id) ON DELETE CASCADE,
    avg_preferred_length    FLOAT,                 -- rolling average of upvoted response lengths
    topic_affinity          JSONB NOT NULL DEFAULT '{}',  -- topic → engagement score
    format_affinity         JSONB NOT NULL DEFAULT '{}',  -- format → satisfaction signal
    query_complexity_avg    FLOAT,                 -- average complexity of user queries
    preferred_response_time TEXT,                  -- 'quick' or 'thorough' inferred
    last_computed_at        TIMESTAMPTZ NOT NULL DEFAULT now(),
    sample_size             INTEGER NOT NULL DEFAULT 0
);

-- Semantic preferences: preference concepts stored as embeddings
CREATE TABLE user_semantic_preferences (
    pref_embedding_id   UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id             UUID NOT NULL REFERENCES users(user_id) ON DELETE CASCADE,
    preference_text     TEXT NOT NULL,             -- human-readable description of the preference
    embedding           VECTOR(1536) NOT NULL,     -- OpenAI text-embedding-3-small or similar
    category            TEXT NOT NULL,             -- 'style', 'domain', 'format', 'persona'
    weight              FLOAT NOT NULL DEFAULT 1.0 CHECK (weight BETWEEN 0.0 AND 5.0),
    source              TEXT NOT NULL DEFAULT 'inferred'
                            CHECK (source IN ('explicit', 'inferred', 'admin')),
    created_at          TIMESTAMPTZ NOT NULL DEFAULT now(),
    expires_at          TIMESTAMPTZ                -- null means permanent
);

CREATE INDEX ON user_semantic_preferences USING hnsw (embedding vector_cosine_ops)
    WITH (m = 16, ef_construction = 64);
CREATE INDEX ON user_semantic_preferences (user_id, category);

-- Preference audit log: every change to explicit preferences is tracked
CREATE TABLE user_preference_audit (
    audit_id        UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id         UUID NOT NULL REFERENCES users(user_id) ON DELETE CASCADE,
    changed_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
    changed_by      TEXT NOT NULL,                 -- 'user', 'system', 'admin'
    field_name      TEXT NOT NULL,
    old_value       TEXT,
    new_value       TEXT,
    session_id      TEXT
);

CREATE INDEX ON user_preference_audit (user_id, changed_at DESC);

-- Row-level security so users can only read their own preferences
ALTER TABLE user_explicit_preferences ENABLE ROW LEVEL SECURITY;
ALTER TABLE user_semantic_preferences ENABLE ROW LEVEL SECURITY;

CREATE POLICY user_own_explicit_prefs ON user_explicit_preferences
    FOR ALL
    USING (user_id = current_setting('app.current_user_id')::UUID);

CREATE POLICY user_own_semantic_prefs ON user_semantic_preferences
    FOR ALL
    USING (user_id = current_setting('app.current_user_id')::UUID);

The separation into four tables is deliberate. Explicit preferences are updated infrequently and queried on every request — they deserve tight typing and a simple unique index. Implicit preferences are computed asynchronously and can be slightly stale; the last_computed_at column lets you decide when to refresh them. Semantic preferences grow over time as the system learns more about a user, so they get their own HNSW index for fast similarity queries.

Capturing and Updating Preferences

You need three update paths: user-initiated changes through an API, system-inferred updates from interaction signals, and the audit trail that connects them.

Start with the FastAPI endpoints for explicit preference management:

# preferences_api.py
from fastapi import FastAPI, Depends, HTTPException, Header
from pydantic import BaseModel, Field
from typing import Optional, List
import asyncpg
import redis.asyncio as aioredis
import json
import uuid
from datetime import datetime

app = FastAPI()

# Connection pool and Redis cache
DB_DSN = "postgresql://appuser:secret@localhost/aiapp"
REDIS_URL = "redis://localhost:6379/0"

# Pydantic models
class ExplicitPreferenceUpdate(BaseModel):
    response_language: Optional[str] = None
    response_length: Optional[str] = Field(None, pattern="^(brief|medium|detailed)$")
    tone: Optional[str] = Field(None, pattern="^(formal|neutral|casual)$")
    code_language: Optional[str] = None
    use_markdown: Optional[bool] = None
    max_response_tokens: Optional[int] = Field(None, ge=50, le=8000)
    domain_focus: Optional[List[str]] = None
    avoid_topics: Optional[List[str]] = None
    custom_instructions: Optional[str] = None

class PreferenceResponse(BaseModel):
    user_id: str
    response_language: str
    response_length: str
    tone: str
    code_language: Optional[str]
    use_markdown: bool
    max_response_tokens: Optional[int]
    domain_focus: Optional[List[str]]
    avoid_topics: Optional[List[str]]
    custom_instructions: Optional[str]
    updated_at: datetime
    version: int

async def get_db():
    conn = await asyncpg.connect(DB_DSN)
    try:
        yield conn
    finally:
        await conn.close()

async def get_redis():
    r = await aioredis.from_url(REDIS_URL, decode_responses=True)
    try:
        yield r
    finally:
        await r.aclose()

async def resolve_user_id(x_user_id: str = Header(...)) -> str:
    """In production this would validate a JWT and extract user_id."""
    return x_user_id

@app.get("/preferences/{user_id}", response_model=PreferenceResponse)
async def get_preferences(
    user_id: str,
    db=Depends(get_db),
    redis=Depends(get_redis),
    current_user: str = Depends(resolve_user_id)
):
    if current_user != user_id:
        raise HTTPException(status_code=403, detail="Access denied")

    # Check cache first
    cache_key = f"prefs:explicit:{user_id}"
    cached = await redis.get(cache_key)
    if cached:
        return PreferenceResponse(**json.loads(cached))

    row = await db.fetchrow(
        """
        SELECT ep.*, u.user_id::text
        FROM user_explicit_preferences ep
        JOIN users u ON u.user_id = ep.user_id
        WHERE ep.user_id = $1
        """,
        uuid.UUID(user_id)
    )
    if not row:
        raise HTTPException(status_code=404, detail="Preferences not found")

    prefs = dict(row)
    prefs["user_id"] = str(prefs["user_id"])
    # Cache for 5 minutes
    await redis.setex(cache_key, 300, json.dumps(prefs, default=str))
    return PreferenceResponse(**prefs)

@app.patch("/preferences/{user_id}")
async def update_preferences(
    user_id: str,
    update: ExplicitPreferenceUpdate,
    db=Depends(get_db),
    redis=Depends(get_redis),
    current_user: str = Depends(resolve_user_id)
):
    if current_user != user_id:
        raise HTTPException(status_code=403, detail="Access denied")

    uid = uuid.UUID(user_id)
    changes = update.model_dump(exclude_none=True)
    if not changes:
        raise HTTPException(status_code=400, detail="No fields to update")

    # Build the audit records and perform the update atomically
    async with db.transaction():
        # Fetch current state for audit
        current = await db.fetchrow(
            "SELECT * FROM user_explicit_preferences WHERE user_id = $1", uid
        )
        if not current:
            raise HTTPException(status_code=404, detail="User preferences not found")

        # Record each changed field in audit log
        audit_records = []
        for field, new_val in changes.items():
            old_val = current[field]
            if old_val != new_val:
                audit_records.append((uid, "user", field, str(old_val), str(new_val)))

        if audit_records:
            await db.executemany(
                """
                INSERT INTO user_preference_audit
                    (user_id, changed_by, field_name, old_value, new_value)
                VALUES ($1, $2, $3, $4, $5)
                """,
                audit_records
            )

        # Dynamic UPDATE — only set fields that were provided
        set_clauses = ", ".join(
            f"{field} = ${i+2}" for i, field in enumerate(changes.keys())
        )
        values = list(changes.values())
        await db.execute(
            f"""
            UPDATE user_explicit_preferences
            SET {set_clauses},
                updated_at = now(),
                version = version + 1
            WHERE user_id = $1
            """,
            uid, *values
        )

    # Invalidate cache
    await redis.delete(f"prefs:explicit:{user_id}")

    # Notify other services via pg_notify
    await db.execute(
        "SELECT pg_notify('preference_updates', $1)",
        json.dumps({"user_id": user_id, "changed_fields": list(changes.keys())})
    )

    return {"status": "updated", "changed_fields": list(changes.keys())}

Now the Celery worker that processes interaction signals and updates implicit preferences asynchronously:

# tasks/preference_inference.py
from celery import Celery
import asyncpg
import asyncio
import json
from anthropic import Anthropic

celery_app = Celery("preference_inference", broker="redis://localhost:6379/1")
anthropic_client = Anthropic()

DB_DSN = "postgresql://appuser:secret@localhost/aiapp"

@celery_app.task(name="infer_preferences_from_interaction")
def infer_preferences_from_interaction(interaction_payload: dict):
    """
    Called after each AI interaction with feedback signals.
    interaction_payload contains: user_id, query, response, rating,
    response_length_chars, topics_detected, format_used
    """
    asyncio.get_event_loop().run_until_complete(
        _async_infer(interaction_payload)
    )

async def _async_infer(payload: dict):
    conn = await asyncpg.connect(DB_DSN)
    try:
        user_id = payload["user_id"]
        rating = payload.get("rating")         # 1-5 or None
        thumbs_up = payload.get("thumbs_up")   # True/False/None
        response_length = payload.get("response_length_chars", 0)
        topics = payload.get("topics_detected", [])

        # Pull recent interaction stats for this user (last 50 interactions)
        stats = await conn.fetchrow(
            """
            SELECT
                COUNT(*) as total,
                AVG(response_length_chars) FILTER (WHERE rating >= 4) as avg_good_length,
                AVG(response_length_chars) FILTER (WHERE thumbs_up = true) as avg_liked_length
            FROM interaction_logs
            WHERE user_id = $1
              AND created_at > now() - interval '30 days'
            """,
            user_id
        )

        # Update topic affinity scores using a simple exponential moving average
        existing = await conn.fetchrow(
            "SELECT topic_affinity, sample_size FROM user_implicit_preferences WHERE user_id = $1",
            user_id
        )

        topic_affinity = json.loads(existing["topic_affinity"]) if existing else {}
        alpha = 0.1  # EMA decay factor

        if thumbs_up is True or (rating and rating >= 4):
            for topic in topics:
                current_score = topic_affinity.get(topic, 0.5)
                topic_affinity[topic] = current_score + alpha * (1.0 - current_score)
        elif thumbs_up is False or (rating and rating <= 2):
            for topic in topics:
                current_score = topic_affinity.get(topic, 0.5)
                topic_affinity[topic] = current_score - alpha * current_score

        # Upsert implicit preferences
        await conn.execute(
            """
            INSERT INTO user_implicit_preferences
                (user_id, avg_preferred_length, topic_affinity, sample_size, last_computed_at)
            VALUES ($1, $2, $3::jsonb, 1, now())
            ON CONFLICT (user_id) DO UPDATE SET
                avg_preferred_length = CASE
                    WHEN $2 IS NOT NULL
                    THEN 0.9 * COALESCE(user_implicit_preferences.avg_preferred_length, $2) + 0.1 * $2
                    ELSE user_implicit_preferences.avg_preferred_length
                END,
                topic_affinity = $3::jsonb,
                sample_size = user_implicit_preferences.sample_size + 1,
                last_computed_at = now()
            """,
            user_id,
            float(response_length) if (thumbs_up or (rating and rating >= 4)) else None,
            json.dumps(topic_affinity)
        )

        # If we have enough data and a strong positive signal, generate a semantic preference
        if thumbs_up is True and stats["total"] >= 10:
            await _maybe_create_semantic_preference(conn, payload)

    finally:
        await conn.close()

async def _maybe_create_semantic_preference(conn, payload: dict):
    """
    Ask Claude to synthesize a semantic preference description from the interaction,
    then store it as a vector.
    """
    from openai import AsyncOpenAI
    oai = AsyncOpenAI()

    prompt = f"""
    A user gave positive feedback on this AI response. 
    Query: {payload.get('query', '')[:500]}
    Response excerpt: {payload.get('response', '')[:500]}
    Topics detected: {', '.join(payload.get('topics_detected', []))}
    Format used: {payload.get('format_used', 'prose')}

    In one sentence, describe what this user seems to prefer about AI responses.
    Be specific. Do not hedge. Return only the sentence.
    """

    message = anthropic_client.messages.create(
        model="claude-opus-4-5",
        max_tokens=100,
        messages=[{"role": "user", "content": prompt}]
    )
    preference_text = message.content[0].text.strip()

    # Embed the preference description
    embedding_response = await oai.embeddings.create(
        model="text-embedding-3-small",
        input=preference_text
    )
    embedding = embedding_response.data[0].embedding

    await conn.execute(
        """
        INSERT INTO user_semantic_preferences
            (user_id, preference_text, embedding, category, source)
        VALUES ($1, $2, $3::vector, 'inferred', 'inferred')
        ON CONFLICT DO NOTHING
        """,
        payload["user_id"],
        preference_text,
        embedding
    )

Injecting Preferences at Inference Time

Stored preferences are worthless if they don't reach the model. The challenge is assembling a complete preference context cheaply on every request. You need all four preference layers — explicit settings, implicit signals, semantically relevant preferences, and conversation history — combined into a coherent system prompt addendum.
