Why self-hosted

Your query plans stay on your server

Most SaaS database tools require you to upload connection strings, query plans, or schema structure to an external service. For regulated industries, security-conscious teams, or air-gapped environments — that's not an option. Every tool here runs completely on your own infrastructure. Nothing leaves your network.

No subscription, no license key, no cost — ever

Install it, run it, use it indefinitely. No seats, no per-query pricing, no surprise invoices, no license phone-home. These tools are completely free. The philosophy is simple: give away good tools, build trust, and let the work speak for itself.

Live data, not stale exports

Static reports go stale. CSV exports miss what happened five minutes ago. Every tool connects directly to your live database and reads from the system catalog, DMVs, and pg_stat_statements — so you always see the current state of your system, not a snapshot from last Tuesday.

Things learned the hard way

01

An index on the right columns in the wrong order does nothing. EXPLAIN will still show an Index Scan. The planner may still pick it. But your sequential page reads triple and your query gets slower, not faster. Composite index column order is the most common indexing mistake I see.

02

VACUUM is not optional. In PostgreSQL, table bloat from dead tuples is silent until it isn't. A table that should be 2GB sitting at 18GB because autovacuum couldn't keep up — that's a performance cliff, not a gradual slide. Monitor it before it matters.

03

The query plan changes when the data changes. A query that ran in 12ms for six months starts taking 4 seconds. Nothing in the code changed. But the table grew from 50k to 5M rows and the planner switched strategies. Statistics staleness is a real production issue, not an edge case.

04

Locking problems look like slow queries. If a query is consistently slow only during business hours and fast at night, look at lock waits before looking at the query plan. The query is fine. Something is holding a lock it shouldn't.

05

Documentation written against a snapshot is wrong the day after migration. Schema docs need to come from a live connection, not an export. The only schema documentation worth trusting is the one generated from the catalog at the moment you read it.

No catch. The tools are completely free — no license key, no subscription, no trial limits, no telemetry. Download, install, and use them indefinitely on your own server. If you find them useful, share them with your team. That's all I ask.
Do the tools work in air-gapped environments?
Yes. The tools run entirely on your own server. The only outbound connection they make is to your database — no telemetry, no analytics, no license phone-home. Suitable for regulated industries, government environments, and air-gapped infrastructure.
What do I need to install the tools?
Basic comfort with a terminal is enough. Each tool ships with a setup script that handles the environment, dependencies, and configuration. You provide your database credentials; the script does the rest. Python 3.10+ and Node.js are the only prerequisites.

Yes, entirely free. Read all three books online, chapter by chapter, no signup required. If you find the content useful, you're more likely to try the tools — that's the model. Give away the knowledge, build trust, let the tools speak for themselves.
