Chapter 19: RAG Evaluation and Feedback Loops Stored in PostgreSQL

Most RAG systems ship with a retrieval pipeline and a generation step, and then the team moves on. Six weeks later, someone notices that the chatbot keeps hallucinating product names, the CEO asks why a competitor's marketing copy shows up in the support bot's answers, and nobody has the data to explain what went wrong. This chapter is about building the instrumentation that prevents that conversation from happening. You will learn how to capture every relevant signal from a RAG pipeline — retrieved chunks, reranking scores, LLM outputs, latency, user feedback — store it all in PostgreSQL, and close the loop so that bad retrievals get fixed automatically. PostgreSQL is the right place for this because evaluation is fundamentally a join problem: you need to correlate user satisfaction signals with the specific chunks that were served, the query that produced them, the embedding model that encoded them, and the generation parameters that shaped the answer. No purpose-built vector store gives you that.

---

19.1 The Evaluation Data Model

Before you can measure anything, you need a schema that captures the full chain of causality in a RAG request. A request is not a single event — it is a pipeline of decisions, each of which can succeed or fail independently. The retrieval step can return irrelevant chunks even when the embedding is good. The generation step can hallucinate even when the retrieval is perfect. The user can mark an answer as unhelpful even when it is technically correct. Your schema has to represent all of these layers without collapsing them together.

The design below uses four core tables that form a natural hierarchy: a session captures a conversation, a query captures a single user turn, a retrieval_result captures each chunk that was fetched for that query, and a generation captures the LLM's response. Feedback and evaluation scores hang off the generation table. This hierarchy means you can answer questions like "which chunks appear most often in low-rated responses" with a single query.

-- Core RAG evaluation schema
-- Run this in your application database

CREATE EXTENSION IF NOT EXISTS pgvector;
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

-- Sessions represent a user's conversation context
CREATE TABLE rag_sessions (
    session_id      UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    user_id         TEXT,
    app_context     TEXT NOT NULL,          -- 'support_bot', 'doc_search', etc.
    metadata        JSONB DEFAULT '{}',
    created_at      TIMESTAMPTZ DEFAULT NOW(),
    ended_at        TIMESTAMPTZ,
    total_queries   INT DEFAULT 0,
    session_rating  SMALLINT CHECK (session_rating BETWEEN 1 AND 5)
);

-- One row per user turn in a session
CREATE TABLE rag_queries (
    query_id            UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    session_id          UUID REFERENCES rag_sessions(session_id),
    query_text          TEXT NOT NULL,
    query_embedding     VECTOR(1536),        -- stored for drift detection
    query_embedding_model TEXT NOT NULL,
    rewritten_query     TEXT,               -- if query rewriting was used
    intent_label        TEXT,               -- classifier output if applicable
    created_at          TIMESTAMPTZ DEFAULT NOW(),
    retrieval_strategy  TEXT NOT NULL,      -- 'semantic', 'hybrid', 'bm25_only'
    top_k               INT NOT NULL,
    similarity_threshold FLOAT,
    retrieval_latency_ms INT,
    generation_latency_ms INT,
    total_latency_ms    INT
);

CREATE INDEX idx_rag_queries_session ON rag_queries(session_id);
CREATE INDEX idx_rag_queries_created ON rag_queries(created_at DESC);
CREATE INDEX idx_rag_queries_embedding ON rag_queries
    USING ivfflat (query_embedding vector_cosine_ops) WITH (lists = 100);

-- Each chunk retrieved for a query, with its ranking context
CREATE TABLE rag_retrieval_results (
    result_id           UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    query_id            UUID REFERENCES rag_queries(query_id) ON DELETE CASCADE,
    chunk_id            UUID NOT NULL,              -- FK to your document chunks table
    document_id         UUID NOT NULL,
    document_title      TEXT,
    chunk_text          TEXT NOT NULL,              -- snapshot at retrieval time
    chunk_embedding     VECTOR(1536),               -- snapshot for offline analysis
    rank_position       INT NOT NULL,               -- 1 = top result
    retrieval_score     FLOAT NOT NULL,             -- cosine similarity or BM25 score
    rerank_score        FLOAT,                      -- cross-encoder score if reranked
    final_rank          INT,                        -- position after reranking
    was_used_in_context BOOLEAN DEFAULT TRUE,       -- false if filtered post-retrieval
    retrieval_method    TEXT NOT NULL               -- 'vector', 'bm25', 'fusion'
);

CREATE INDEX idx_retrieval_query ON rag_retrieval_results(query_id);
CREATE INDEX idx_retrieval_chunk ON rag_retrieval_results(chunk_id);
CREATE INDEX idx_retrieval_document ON rag_retrieval_results(document_id);

-- The LLM's response and associated metadata
CREATE TABLE rag_generations (
    generation_id       UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    query_id            UUID REFERENCES rag_queries(query_id) ON DELETE CASCADE,
    model_name          TEXT NOT NULL,              -- 'claude-3-5-sonnet-20241022'
    model_provider      TEXT NOT NULL,              -- 'anthropic', 'openai'
    prompt_tokens       INT,
    completion_tokens   INT,
    total_tokens        INT,
    response_text       TEXT NOT NULL,
    finish_reason       TEXT,                       -- 'stop', 'max_tokens', etc.
    temperature         FLOAT,
    system_prompt_hash  TEXT,                       -- SHA256 of system prompt
    created_at          TIMESTAMPTZ DEFAULT NOW(),
    -- Automated evaluation scores (filled async)
    faithfulness_score  FLOAT CHECK (faithfulness_score BETWEEN 0 AND 1),
    relevance_score     FLOAT CHECK (relevance_score BETWEEN 0 AND 1),
    context_recall      FLOAT CHECK (context_recall BETWEEN 0 AND 1),
    answer_correctness  FLOAT CHECK (answer_correctness BETWEEN 0 AND 1),
    eval_model          TEXT,                       -- model used for evaluation
    eval_completed_at   TIMESTAMPTZ
);

CREATE INDEX idx_generation_query ON rag_generations(query_id);
CREATE INDEX idx_generation_model ON rag_generations(model_name);
CREATE INDEX idx_generation_scores ON rag_generations(faithfulness_score, relevance_score);

-- User feedback: thumbs, ratings, corrections
CREATE TABLE rag_feedback (
    feedback_id         UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    generation_id       UUID REFERENCES rag_generations(generation_id),
    query_id            UUID REFERENCES rag_queries(query_id),
    user_id             TEXT,
    feedback_type       TEXT NOT NULL,     -- 'thumbs_up', 'thumbs_down', 'rating', 'correction'
    rating              SMALLINT CHECK (rating BETWEEN 1 AND 5),
    comment             TEXT,
    corrected_answer    TEXT,              -- user-provided ground truth
    flagged_chunk_ids   UUID[],            -- chunks the user identified as wrong
    created_at          TIMESTAMPTZ DEFAULT NOW(),
    reviewed_by         TEXT,             -- human reviewer if escalated
    review_action       TEXT,             -- 'accepted', 'rejected', 'escalated'
    review_notes        TEXT
);

CREATE INDEX idx_feedback_generation ON rag_feedback(generation_id);
CREATE INDEX idx_feedback_created ON rag_feedback(created_at DESC);
CREATE INDEX idx_feedback_type ON rag_feedback(feedback_type);

-- Aggregate daily metrics for dashboards (populated by scheduled job)
CREATE TABLE rag_daily_metrics (
    metric_date         DATE NOT NULL,
    app_context         TEXT NOT NULL,
    model_name          TEXT NOT NULL,
    retrieval_strategy  TEXT NOT NULL,
    total_queries       INT DEFAULT 0,
    avg_faithfulness    FLOAT,
    avg_relevance       FLOAT,
    avg_context_recall  FLOAT,
    thumbs_up_count     INT DEFAULT 0,
    thumbs_down_count   INT DEFAULT 0,
    avg_retrieval_ms    FLOAT,
    avg_generation_ms   FLOAT,
    p95_total_ms        FLOAT,
    PRIMARY KEY (metric_date, app_context, model_name, retrieval_strategy)
);

19.2 Instrumenting the RAG Pipeline

The schema is only useful if the pipeline actually writes to it. The instrumentation layer needs to add minimal overhead to the request path — write the synchronous data inline, and dispatch the expensive evaluation work asynchronously. The pattern below uses a RAGPipeline class that wraps your retrieval and generation calls, writes the core rows synchronously, and enqueues an evaluation task via Celery.

# rag_pipeline.py — fully instrumented RAG pipeline
# pip install anthropic openai psycopg2-binary celery pgvector numpy

import hashlib
import time
import uuid
from datetime import datetime, timezone
from typing import Optional

import anthropic
import numpy as np
import psycopg2
import psycopg2.extras
from celery import Celery
from openai import OpenAI

psycopg2.extras.register_uuid()

# --- Configuration ---
DB_DSN = "postgresql://app_user:secret@localhost:5432/rag_app"
CELERY_BROKER = "redis://localhost:6379/0"

celery_app = Celery("rag_eval", broker=CELERY_BROKER, backend=CELERY_BROKER)
openai_client = OpenAI()
anthropic_client = anthropic.Anthropic()


def get_db():
    return psycopg2.connect(DB_DSN)


def embed_text(text: str, model: str = "text-embedding-3-small") -> list[float]:
    response = openai_client.embeddings.create(input=text, model=model)
    return response.data[0].embedding


def retrieve_chunks(
    query_embedding: list[float],
    top_k: int = 5,
    similarity_threshold: float = 0.75,
    strategy: str = "semantic",
) -> list[dict]:
    """
    Your actual retrieval logic against the document_chunks table.
    Returns list of dicts with chunk_id, document_id, chunk_text, score, etc.
    This is simplified — swap in your real retrieval query.
    """
    conn = get_db()
    cur = conn.cursor(cursor_factory=psycopg2.extras.DictCursor)
    embedding_str = "[" + ",".join(str(x) for x in query_embedding) + "]"
    cur.execute(
        """
        SELECT
            chunk_id,
            document_id,
            document_title,
            chunk_text,
            1 - (embedding <=> %s::vector) AS score
        FROM document_chunks
        WHERE 1 - (embedding <=> %s::vector) >= %s
        ORDER BY embedding <=> %s::vector
        LIMIT %s
        """,
        (embedding_str, embedding_str, similarity_threshold, embedding_str, top_k),
    )
    rows = cur.fetchall()
    cur.close()
    conn.close()
    return [dict(r) for r in rows]


class RAGPipeline:
    def __init__(self, app_context: str, model: str = "claude-3-5-sonnet-20241022"):
        self.app_context = app_context
        self.model = model
        self.embedding_model = "text-embedding-3-small"

    def run(
        self,
        session_id: str,
        user_id: str,
        query_text: str,
        top_k: int = 5,
        temperature: float = 0.2,
    ) -> dict:
        query_id = str(uuid.uuid4())
        conn = get_db()

        try:
            # 1. Embed the query
            t0 = time.time()
            query_embedding = embed_text(query_text, model=self.embedding_model)
            embed_ms = int((time.time() - t0) * 1000)

            # 2. Retrieve chunks
            t1 = time.time()
            chunks = retrieve_chunks(query_embedding, top_k=top_k)
            retrieval_ms = int((time.time() - t1) * 1000)

            # 3. Write the query row (synchronous — fast)
            embedding_str = "[" + ",".join(str(x) for x in query_embedding) + "]"
            with conn.cursor() as cur:
                cur.execute(
                    """
                    INSERT INTO rag_queries (
                        query_id, session_id, query_text, query_embedding,
                        query_embedding_model, retrieval_strategy, top_k,
                        retrieval_latency_ms
                    ) VALUES (%s, %s, %s, %s::vector, %s, %s, %s, %s)
                    """,
                    (
                        query_id,
                        session_id,
                        query_text,
                        embedding_str,
                        self.embedding_model,
                        "semantic",
                        top_k,
                        retrieval_ms,
                    ),
                )

                # 4. Write retrieval results
                for rank, chunk in enumerate(chunks, start=1):
                    cur.execute(
                        """
                        INSERT INTO rag_retrieval_results (
                            query_id, chunk_id, document_id, document_title,
                            chunk_text, rank_position, retrieval_score,
                            was_used_in_context, retrieval_method
                        ) VALUES (%s, %s, %s, %s, %s, %s, %s, %s, %s)
                        """,
                        (
                            query_id,
                            chunk["chunk_id"],
                            chunk["document_id"],
                            chunk.get("document_title", ""),
                            chunk["chunk_text"],
                            rank,
                            chunk["score"],
                            True,
                            "vector",
                        ),
                    )
                conn.commit()

            # 5. Build context and call LLM
            context = "\n\n---\n\n".join(c["chunk_text"] for c in chunks)
            system_prompt = (
                "You are a helpful assistant. Answer the user's question using "
                "only the context provided. If the context does not contain enough "
                "information, say so explicitly. Do not make up information."
            )
            system_hash = hashlib.sha256(system_prompt.encode()).hexdigest()[:16]

            t2 = time.time()
            response = anthropic_client.messages.create(
                model=self.model,
                max_tokens=1024,
                temperature=temperature,
                system=system_prompt,
                messages=[
                    {
                        "role": "user",
                        "content": f"Context:\n{context}\n\nQuestion: {query_text}",
                    }
                ],
            )
            generation_ms = int((time.time() - t2) * 1000)

            response_text = response.content[0].text
            generation_id = str(uuid.uuid4())

            # 6. Write generation row
            with conn.cursor() as cur:
                cur.execute(
                    """
                    INSERT INTO rag_generations (
                        generation_id, query_id, model_name, model_provider,
                        prompt_tokens, completion_tokens, total_tokens,
                        response_text, finish_reason, temperature, system_prompt_hash
                    ) VALUES (%s, %s, %s, %s, %s, %s, %s, %s, %s, %s, %s)
                    """,
                    (
                        generation_id,
                        query_id,
                        self.model,
                        "anthropic",
                        response.usage.input_tokens,
                        response.usage.output_tokens,
                        response.usage.input_tokens + response.usage.output_tokens,
                        response_text,
                        response.stop_reason,
                        temperature,
                        system_hash,
                    ),
                )

                # Update latency totals on the query row
                total_ms = embed_ms + retrieval_ms + generation_ms
                cur.execute(
                    """
                    UPDATE rag_queries
                    SET generation_latency_ms = %s, total_latency_ms = %s
                    WHERE query_id = %s
                    """,
                    (generation_ms, total_ms, query_id),
                )
                conn.commit()

            # 7. Enqueue async evaluation — does NOT block the response
            evaluate_generation.delay(
                generation_id=generation_id,
                query_text=query_text,
                response_text=response_text,
                context_chunks=[c["chunk_text"] for c in chunks],
            )

            return {
                "query_id": query_id,
                "generation_id": generation_id,
                "answer": response_text,
                "sources": [
                    {
                        "title": c.get("document_title"),
                        "score": c["score"],
                        "rank": i + 1,
                    }
                    for i, c in enumerate(chunks)
                ],
                "latency_ms": total_ms,
            }

        finally:
            conn.close()


@celery_app.task(bind=True, max_retries=3)
def evaluate_generation(
    self,
    generation_id: str,
    query_text: str,
    response_text: str,
    context_chunks: list[str],
):
    """
    Async task: uses an LLM-as-judge to score the generation.
    Runs after the user has already received their answer.
    """
    try:
        scores = run_llm_evaluation(query_text, response_text, context_chunks)
        conn = get_db()
        with conn.cursor() as cur:
            cur.execute(
                """
                UPDATE rag_generations SET
                    faithfulness_score   = %s,
                    relevance_score      = %s,
                    context_recall       = %s,
                    answer_correctness   = %s,
                    eval_model           = %s,
                    eval_completed_at    = NOW()
                WHERE generation_id = %s
                """,
                (
                    scores["faithfulness"],
                    scores["relevance"],
                    scores["context_recall"],
                    scores["answer_correctness"],
                    "gpt-4o-mini",
                    generation_id,
                ),
            )
            conn.commit()
        conn.close()
    except Exception as exc:
        raise self.retry(exc=exc, countdown=60)


def run_llm_evaluation(
    query: str, answer: str, context_chunks: list[str]
) -> dict[str, float]:
    """
    LLM-as-judge evaluation using GPT-4o-mini for cost efficiency.
    Returns scores between 0 and 1 for each dimension.
    """
    context = "\n\n".join(context_chunks)
    eval_prompt = f"""You are evaluating a RAG system response. Score each dimension from 0.0 to 1.0.

QUESTION: {query}

CONTEXT PROVIDED TO THE SYSTEM:
{context}

SYSTEM'S ANSWER:
{answer}

Score the following and respond with ONLY a JSON object, no explanation:
- faithfulness: Is every claim in the answer supported by the context? (1.0 = fully grounded, 0.0 = hallucinated)
- relevance: Does the answer actually address the question? (1.0 = directly answers, 0.0 = off-topic)
- context_recall: Does the answer use the most relevant information from the context? (1.0 = all key facts used)
- answer_correctness: Overall quality combining faithfulness and relevance.

Example response format:
{{"faithfulness": 0.95, "relevance": 0.88, "context_recall": 0.72, "answer_correctness": 0.85}}"""

    response = openai_client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": eval_prompt}],
        temperature=0.0,
        response_format={"type": "json_object"},
    )

    import json
    scores = json.loads(response.choices[0].message.content)
    # Clamp to [0, 1]
    return {k: max(0.0, min(1.0, float(v))) for k, v in scores.items()}

19.3 Feedback Collection and User Signal Processing

Automated evaluation tells you what the model thinks of its own outputs. User feedback tells you what your users think, which is a different and more important signal. The two signals rarely agree perfectly — a user might rate an answer highly even when the faithfulness score is low, because the answer was phrased confidently. That disagreement is itself valuable data.
